Abstract Data Types: Array and Structure

Posted on 2019-03-04 | In Data Structure | |

Many junior programmers view array as a continuous memory area, which included me. Truth is, that is not all of it, merely the implementation part. Intuitively, every index of the array should have one and only one corresponding value. This process is called mapping (or correspondence) and the corresponding value is the data object. For example, myDataObject = {apple, chair, 2, 5.2, red, green, Jack} Note that these objects may or may not be related.

Now let’s look upon the implementation of Abstract Data Type (ADT) array. The compiler allocate certain consecutive store units. For example, the results of the following code are

#include <stdio.h>

void printLocation(int *p1, int cols)
{

    int i;

    printf("The address of every elements are\n");
    for (i=0; i<cols; i++)
    printf("%X%10d\n",p1+i,*(p1+i));
    printf("\n");

}


void main()
{


    int Array_1[5]= {1,2,3,4,5};

    printLocation(&Array_1[0],sizeof(Array_1)/sizeof(int));

    printf("%d\n",sizeof(Array_1));


}

Note that every data object is allocated 4 spaces in memory shown by the pointer’s value. Another thing to remember is that the function sizeof() in C is different than in MATLAB where the same function returns 5 (objects) instead of 20 (5*4 memory units for mapping).

So, now we knew that to represent a two-dimensional array, say 10 times 10, we will need 100 times sizeof(ADT) memory units. However, this could be reduced if the majority of the matrix is 0, or as we call, the Sparse Matrix.

It is a waste of resources to store 0, as the information entropy of the event that 0 appears in the matrix is relatively low. In this case, we only consider the non-zero items and mark down their postion (rows & cols). For example, if we consider a 6 by 6 matrix with 8 non-zero items like this,

A =    [0 0 1 0 0 0
    0 3 4 0 0 5
    0 0 0 0 0 6
    0 0 8 0 0 0
    0 9 0 7 0 0
    0 0 0 0 0 0 ]

It could be rewrite as such

    rows    cols    values
a[0]    6        6        7
a[1]    0        2        1
a[2]    1        1        3
a[3]    1        2        4
a[4]    1        5        5
a[5]    2        5        6
a[6]    3        5        8
a[7]    4        1        9
a[8]    4        3        7

As we can see here, we list the location and values as well as overall size and total number of non-zeros items, in the order of rows. In this case, we only use 3 times 9 =27 items instead of 6 times 6 =36.

This method is also good for transposition of matrixs. Using the traditional storing method, we need to iterate and visit every items in the matrixs. But now, with mere a re-arrange in the order of cols sorting, instead of rows, and regard the cols’ value as new rows’ index, the transposition procedure will be done.
